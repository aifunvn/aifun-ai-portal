Slide 6 — Bài học kỹ thuật #1
CORS với Google Apps Script
Điểm kỹ thuật quan trọng nhất khi tích hợp GAS với bất kỳ frontend nào ngoài Google.
❌ Sai — gây CORS error
Dùng Content-Type: application/json
→ Browser tự động gửi OPTIONS preflight
→ GAS không handle được OPTIONS request
→ Browser block → "Failed to fetch"
✅ Đúng — hoạt động
Dùng Content-Type: text/plain
→ Browser coi đây là "simple request"
→ Không gửi OPTIONS preflight
→ GAS nhận POST bình thường → Thành công
// ✅ Code đúng trong skill-engine.js
fetch(GAS_URL, {
  method: 'POST',
  headers: { 'Content-Type': 'text/plain' },  // ← quan trọng
  body: JSON.stringify(payload)
})
⚠️ Lưu ý: GAS tự thêm Access-Control-Allow-Origin: * vào response — nhưng chỉ khi request là "simple request". Không dùng mode: 'no-cors' vì sẽ không đọc được response.
Slide 7 — Bài học kỹ thuật #2
Truy cập Google Sheets từ GAS
openById() có thể fail silently — gây ra lỗi khó debug nhất trong dự án này.
❌ Dễ lỗi — openById
SpreadsheetApp.openById(id)
→ Lỗi "Illegal spreadsheet id" khi ID bị truncate hoặc account permissions
→ Nằm trong try-catch → fail silently
→ Document vẫn trả về success nhưng không ghi Sheets
✅ Ổn định — DriveApp
DriveApp.getFilesByName('AIFUN DATA CENTER')
→ Tìm file theo tên trong Drive của owner
→ SpreadsheetApp.open(file)
→ Không phụ thuộc vào Spreadsheet ID cứng
// ✅ Code đúng trong Logger.gs và Mã.gs
var files = DriveApp.getFilesByName('AIFUN DATA CENTER');
if (!files.hasNext()) { Logger.log('Không tìm thấy file'); return; }
var ss = SpreadsheetApp.open(files.next());
var sheet = ss.getSheetByName('GENERATED_DOCUMENTS');
sheet.appendRow([id, skillTitle, title, docUrl, date, user, prompt, status]);
⚠️ Quan trọng: Deploy GAS phải chọn "Execute as: Tôi (owner)". Nếu chọn "Anyone" → GAS chạy với quyền người gọi → không thấy file trong Drive của owner.

Slide 9 — Hướng dẫn vận hành
Checklist debug khi có sự cố
Kiểm tra theo đúng thứ tự này — tiết kiệm 80% thời gian debug.
  • 1
    Kiểm tra GAS URL trong config.js
    Vào GAS → Triển khai → Quản lý triển khai → Copy URL → So sánh với gasWebAppUrl trong config.js. Nếu khác → cập nhật config.js → push lên GitHub.
  • 2
    Chạy test function trong GAS Editor
    Mở GAS Editor → chọn hàm testSaveSheets → Run → xem Logs. Nếu success → GAS OK, lỗi nằm ở frontend. Nếu lỗi → fix GAS trước.
  • 3
    Mở Console browser (F12) khi bấm nút
    "Failed to fetch" = CORS → kiểm tra Content-Type. "undefined is not a function" = JS file chưa load đúng. Đọc stack trace để xác định file và dòng lỗi.
  • 4
    Deploy lại GAS sau mỗi lần sửa code
    Sửa bất kỳ file .gs nào → bắt buộc phải: Triển khai → Quản lý → Chỉnh sửa → Phiên bản mới → Triển khai. Không deploy = code cũ vẫn chạy.
  • 5
    Hard refresh sau khi push GitHub
    GitHub Pages cache ~2 phút. Dùng Ctrl+Shift+R để force reload. Vẫn thấy cũ → đợi thêm 2 phút → thử lại. Kiểm tra tab Network trong DevTools để xác nhận file mới.
Slide 10 — Tổng kết & Bước tiếp theo
Bài học & Roadmap phát triển
Những insight quan trọng nhất — áp dụng cho mọi dự án tích hợp AI tiếp theo.
Bài học 1
Test từng layer riêng biệt
Luôn có hàm test độc lập trong GAS (testSaveSheets). Xác định lỗi ở frontend hay backend trước khi debug toàn luồng — tiết kiệm hàng giờ.
Bài học 2
CORS là điểm nghẽn phổ biến
Bất kỳ API nào không hỗ trợ OPTIONS preflight → dùng Content-Type: text/plain. Nguyên tắc này áp dụng cho GAS và nhiều API khác.
Bài học 3
Silent fail nguy hiểm hơn crash
try-catch nuốt lỗi mà không báo → hệ thống trông như chạy đúng nhưng không làm gì. Luôn log error và trả về error trong response để frontend biết.
